          腾讯御安全脱壳与修复

          duanmulixiang 2021-02-05 脱壳

          # 准备

          1. 逍遥模拟器（因为支持xposed installer）
          2. xposed install （用来支持反射大师）
          3. mt管理器（免费版即可,用了签名和替换dex）
          4. 反射大师清羽版（用来脱出dex）
          5. jadx-gui （用来查看dex的java代码）
          6. 安卓逆向助手（用来修复dex,反编译dex,重打包dex）

          使用到的工具地址

          # 开始

          模拟器安装好xposed，mt,反射大师，某腾讯御安全加固的apk,打开反射大师选取apk,然后打开apk，有的软件需要多点击几个页面不然可能脱不全，反射大师选择提取dex,记下路径，打开mt管理器，提取待脱壳的apk,打开apk,修改AndroidManifest.xml,找到application节点里面的android:name,这里是应用的入口，御安全的是android:name="MyWrapperProxyApplication",使用jadx打开apk,查看这个类的代码，可以看到定义了className的局部变量，这就是应用的真正入口，将android:name修改为这个即可。然后打开安卓逆向助手，选择修复dex(因为mt管理器会员才能修复，所以使用逆向助手来修复，必须要做否则可能无法启动应用)，然后反编译dex得到smali文件,使用jadx打开dex查看应用的逻辑，对照着修改smali的内容，修改完之后使用逆向助手重新打包dex,将dex放到模拟器的共享文件夹，使用mt管理器打开apk,删除掉原来的dex,可能有多个，那么替换过去的dex也对着修改名称，通常只有一个叫做classes.dex，将修改好的dex换进去，这些步骤都不要签名,最后退出来重新签名安装即可。

          # 注意

          步骤不难，有几个注意的点。

          1. 对于修改跳转逻辑的，不一定需要修改if-nez v0, :cond_d这种条件，可以直接使用goto :cond_d跳转过去。
          2. 有时候只能在第一次进入apk的时候才能提取dex。
          3. 对于增加逻辑的时候，如果要新建变量，尽量新建一个方法来调用，这样不会影响原来的寄存器大小，不然修改起来很麻烦。 比如需要将广告的view去掉的时候，可以新增一个方法
          4. 寄存器个数+1的时候，接着v变量接着使用，因为p变量始终占据最后几个寄存器。比如有3个寄存器v0 p0 p1,修改.registers 4,这是应该使用v1而不是p2.

          # 寄存器的命名

          v命名法采用小写的’v‘开头，表示函数中用到的局部变量与参数，所有的寄存器命名从v0开始，依次递增。p命名法采用小写的p开头，其命名规则是：函数中引入的参数命名从p0开始，依次递增。 假设一个函数中使用到了M个寄存器，并且该函数有N个参数，根据Dalvik虚拟机参数传递方式中的规定：参数使用最后的N个寄存器中，局部变量使用从v0开始的前M-N个寄存器。下面给出一副图示说明：
